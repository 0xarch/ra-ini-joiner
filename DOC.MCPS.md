# 宏

在所有小节中，以 `@` 开头的键值对都会被视作是宏。这可以通过 `Macro.IgnoreKeys` 拒绝。你也可以通过 `Macro.ExplicitKeys` 显式指定哪些语句是宏.

与宏处理相关的日志以 `[MCPS]` 开头。

## 调用

宏的调用语法包括以下几种：

```ini
# 标准：INI字符串调用
@File:Foo=Bar

# 标准：INI数组调用
@File:Foo[]=Bar1
@File:Foo[]=Bar2

# 糖  ：INI伪数组调用
@File:Foo:=Bar1,Bar2

# 糖  ：直接INI字符串调用
@Foo=Bar

# 糖  ：直接INI数组调用
@Foo[]=Bar1
@Foo[]=Bar2

# 糖  ：直接INI伪数组调用
@Foo:=Bar1,Bar2
```

大致可以认为是不同的键书写方式与值书写方式的组合。因此下面只列出这些键和值的书写方式。

值得注意的是，V2的宏接管了继承功能。在V1中，你需要通过`@Inherits=文件名:小节名`来特殊地指定，而现在，你可以直接使用 `@文件名:小节名` 来告知处理器将那个小节复制过来。这也同时实现了多继承支持。目前，我们使用的 INI 解析器仍然可以正确解析。

`文件名` 按如下机制解析：首先，认为其是以**该文件**为目录的相对路径，若索引中没有发现目标文件，那么认为其是以`Macro.Root`为目录的相对路径，如果没有后缀名则会添加`Macro.DefaultFileSuffix`，该值默认为`.ini`。

### 键

* 标准：`@文件名:小节名`，表示将 `文件名` 文件中 `小节名` 宏拼接到该位置。
* 糖： `@小节名`，表示将 `小节名` 文件中 `小节名` 宏拼接到该位置。即文件名与小节名相同。

### 值

* 字符串： `=值`，正常写法。
* 数组：`[]=值1; []=值2; ...`，正常数组写法。
* 伪数组： `:=值1,值2,...`，这是不被广泛支持的扩展写法。值将被按 `,` 分割后转化为数组。注意无法区分分隔符是否是值内容，不会将 `"` 包裹的内容解析为字符串。

## 定义

你可以在任何文件中定义宏。一个小节是否是宏取决于他是否被当作宏调用了。
你可以在小节里添加 `@@DontExportMe` 键来阻止一个在普通文件中定义的小节被输出。

宏小节在语法上与其他小节并无不同。然而，宏小节支持使用参数批量生成键或值。

### 使用 JavaScript 作为宏

你可以使用 `.mjs` 或 `.js` 文件作为 JavaScript 宏文件，这些文件强制拒绝被输出。你需要为每个 JavaScript 宏文件定义一个默认导出，这等同于 `文件名:文件名`，其他具名导出则等同于 `文件名:函数名`。当然，`default`本身仍然可用。
